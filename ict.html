SESSION 1 - Overview of OverTheWire and UnderTheWire

Fantastic war game platforms that teach hands-on Linux, networking, and Windows command line skills in a Capture The Flag (CTF)-like format.

From OverTheWire (Linux) skills reinforced:

  • Terminal basics - ls, cd to list files and change directories
  • Reading/manipulating files – cat, file
  • Recognizing file encoding – Base64
  • Dealing with special characters – ./-filename
  • SSH usage – to connect remotely

From UnderTheWire (PowerShell) skills reinforced:

  • PowerShell scripting
  • Windows file system and registry
  • Object-oriented shell concepts
  • String manipulation
  • System enumeration

SESSION 2 - Networking Fundamentals

1. DNS (Domain Name System)

DNS translates human-readable domain names (like google.com) into IP addresses (like 142.250.190.14).

  • Pinging is used to test connectivity and resolve hostnames to IP addresses: ping google.com
  • View DNS server configuration on Linux: cat /etc/resolv.conf
  • BGP Tools (Border Gateway Protocol): Check routing paths and DNS propagation.
  • DNS as attack/survey vectors: Subdomain enumeration, zone transfers, DNS poisoning/spoofing

2. OSI Model (Open Systems Interconnection)

A 7-layer model explaining how data flows in a network:

  • 1. Physical – Hardware (cables, NICs, signals)
  • 2. Data Link – MAC addressing, switches
  • 3. Network – IP addressing, routing
  • 4. Transport – TCP/UDP
  • 5. Session – Controls connections
  • 6. Presentation – Data formatting, encryption
  • 7. Application – User interfaces (HTTP, DNS)

TCP/IP Model simplifies to 4 layers: Link, Internet, Transport, Application.

3. Pyramid of Pain

A cybersecurity model showing difficulty of defending against different indicators of compromise (IOCs).

  • Hash Values – Easily changed
  • IP Addresses – Switched frequently
  • Domain Names – Slightly harder to change
  • Network/Host Artifacts – Patterns, configs
  • Tools – Specific attack tools
  • TTPs – Attacker behavior & strategy

4. Active Directory

  • User Authentication: verifies usernames and passwords
  • Resource Management: controls access
  • Group Policy: pushes security settings

SESSION 3 - Offensive and Defensive Security

1. Offensive Security

Simulating attacks to identify vulnerabilities.

  1. Reconnaissance – whois, nslookup
  2. Scanning & Enumeration – nmap, netcat
  3. Exploitation – Metasploit
  4. Post-Exploitation – mimikatz

2. Defensive Security

  1. Monitoring – SIEMs like Splunk
  2. Log Analysis – Sysmon
  3. Detection – IDS/IPS like Snort
  4. Incident Response – Playbooks, forensics
  5. Threat Intelligence – CTI platforms, MITRE ATT&CK
