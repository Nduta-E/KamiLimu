GitHub and Contributing to Open Source Projects

Why GitHub?

  • Git needed a graphical user interface
  • En masse source code sharing
  • Efficient communication and collaboration
  • GitHub is free!

Basic Workflow:

  • Use of fork to create a copy of another repository (e.g., forked the KamiLimu repo to my own page)
  • Use of git pull to pull current updates from the KamiLimu repo
  • git status – check status of the repo
  • git add – add the updated file to the repo
  • git commit – commit the changes of the added file to the repo
  • git push – push the committed changes to the forked repo

Once pushed, the changes go as a merge request, and when accepted, you become a contributor to the project. This worked as a perfect example of contributing to open source projects.
